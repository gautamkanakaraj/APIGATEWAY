# Architecture Brief: Telemetry-Enabled API Gateway

This document provides a technical overview of the API Gateway, detailing its design, core pillars, and downstream routing mechanisms.

---

## 🏗️ Core Architectural Pillars

### 1. Ingress & Routing Layer (Go)
- **Role**: Serves as the entrance for all edge request traffic.
- **Routing**: Matches path prefixes (e.g., `/api/v1/products`) to backend server pools using an lock-free round-robin algorithm.
- **Reverse Proxy**: Utilizes Go's built-in `httputil.ReverseProxy` to stream client requests downstream, automatically preserving body payloads and response statuses.

### 2. Distributed Rate Limiter (Redis + Lua Script)
- **Algorithm**: Token Bucket.
- **Implementation**: Runs atomically inside Redis via a Lua script (`limiter/limiter.go`) to prevent race conditions during concurrent requests.
- **Precision**: Millisecond-level resolution. Token replenishment is calculated at the millisecond scale (tokens filled per millisecond = `fill_rate / 1000`), ensuring smooth refilling for sub-second traffic profiles.
- **Tiers**: Looked up dynamically from `gateway.yaml` based on the caller's JWT claims:
  - `premium`: High burst capacity and fill rate.
  - `free`: Low capacity and fill rate, short-circuiting requests with `429 Too Many Requests` once depleted.

### 3. Fault Tolerance & Circuit Breaking (`sony/gobreaker`)
- **Isolation**: Each backend target maintains its own individual Circuit Breaker instance.
- **Thresholds**: If a backend's error rate exceeds 50% over a window of 5 or more requests, the breaker trips to **Open** state.
- **Failover**: The round-robin load balancer detects open breakers and bypasses degraded instances, routing traffic to healthy alternatives without impacting client success.
- **Cooldown**: A 5-second cooldown period allows trial traffic under a **Half-Open** state to verify if the backend service has recovered.

### 4. Zero-Downtime Hot-Reloading (`fsnotify`)
- **Event Watcher**: A background goroutine watches the configuration directory using file system notifications.
- **Thread Safety**: Updates to `gateway.yaml` instantly trigger a configuration parse. The gateway acquires a write lock, updates the route mappings and tier limits, cancels old health checking daemons, and starts new ones without needing a process restart.

### 5. Real-Time Observability Control Board
- **Backend Logging Stream**: Connects to the local Docker socket (`/var/run/docker.sock`) over HTTP. It reads the container log stream, automatically parses multiplexed data, and broadcasts structured events to frontend clients.
- **Server-Sent Events (SSE)**: Streams proxy pass alerts, rate limiting blocks (429s), circuit breaker state changes, and service health states to clients in real-time.
- **Frontend Dashboard**: Visualizes traffic topology, dynamic token bucket capacities, request distribution ratios, and service statuses inside a translucent, glassmorphic dark UI.
